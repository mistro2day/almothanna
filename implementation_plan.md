# خطة نقل البيانات من النظام القديم (MSSQL) إلى النظام الجديد (PostgreSQL)

توضح هذه الخطة خطوات تثبيت الحزم المطلوبة (`drizzle-kit` و `mssql`) وتهيئة بيئة العمل لنقل البيانات من قاعدة بيانات النظام القديم (الموجودة كملف نسخة احتياطية `.bak` باسم `MicroPOS_2026-06-01_17-17-46.bak`) إلى قاعدة بيانات PostgreSQL الحالية باستخدام Prisma ORM.

## مراجعة المستخدم المطلوبة

> [!IMPORTANT]
> بما أن أداة تشغيل الأوامر (Terminal) معطلة حالياً، يرجى تشغيل الأوامر الموضحة في كتل الأكواد يدوياً في مجلد `backend` الخاص بك.

> [!WARNING]
> يتطلب استيراد البيانات من ملف النسخة الاحتياطية `.bak` استعادة قاعدة البيانات أولاً على خادم SQL Server محلي أو سحابي لتتمكن أداة النقل من الاتصال بها وقراءة البيانات.

## الأسئلة المفتوحة
- هل قمت بالفعل باستعادة قاعدة البيانات من ملف `MicroPOS_2026-06-01_17-17-46.bak` على خادم SQL Server محلي؟ إذا كانت الإجابة نعم، يرجى تزويدنا ببيانات الاتصال (المضيف، المنفذ، اسم المستخدم، وكلمة المرور).
- ما هي الجداول الأساسية التي ترغب في نقل بياناتها أولاً؟ (مثل: المنتجات، الموردين، العملاء، إلخ).

## جدول الأقسام المراد تنفيذها

| القسم | الوصف | الحالة |
| :--- | :--- | :--- |
| **1. تثبيت الحزم** | تثبيت `drizzle-kit` و `mssql` و `@types/mssql` في مجلد backend | بانتظار الموافقة |
| **2. إعداد الاتصال** | إضافة متغيرات بيئة الاتصال بـ MSSQL في ملف `.env` | بانتظار الموافقة |
| **3. سكربت الهجرة (Migration Script)** | إنشاء سكربت TypeScript للاتصال بـ SQL Server ونقل البيانات إلى PostgreSQL عبر Prisma | بانتظار الموافقة |
| **4. التشغيل والتحقق** | تشغيل السكربت والتحقق من صحة نقل البيانات | بانتظار الموافقة |

## التغييرات المقترحة

### [تهيئة البيئة وتثبيت الحزم]

يرجى تشغيل الأوامر التالية في موجّه الأوامر (Terminal) الخاص بك داخل مجلد المشروع:

```bash
# الانتقال إلى مجلد backend
cd backend

# تثبيت الحزم المطلوبة
npm install -D drizzle-kit @types/mssql
npm install mssql
```

### [تعديل ملف البيئة]

#### [MODIFY] [.env](file:///d:/almothanna/backend/.env)
إضافة متغيرات الاتصال بقاعدة البيانات القديمة:
```env
MSSQL_HOST="localhost"
MSSQL_PORT=1433
MSSQL_USER="sa"
MSSQL_PASSWORD="YourPasswordHere"
MSSQL_DATABASE="MicroPOS"
```

---

### [إنشاء سكربت نقل البيانات]

#### [NEW] [migrate-old-db.ts](file:///d:/almothanna/backend/prisma/migrate-old-db.ts)
سنقوم بإنشاء سكربت مخصص للاتصال بـ SQL Server وقراءة البيانات ثم إدخالها في PostgreSQL باستخدام Prisma Client.

---

## خطة التحقق

### التحقق اليدوي
1. تشغيل السكربت باستخدام `npx ts-node prisma/migrate-old-db.ts`.
2. فحص السجلات (Logs) للتأكد من نقل السجلات دون أخطاء.
3. فحص قاعدة البيانات الجديدة للتأكد من مطابقة عدد السجلات.
